# דפוסי משימה אותנטיים (קצרים) — עברית

מסמך תכנון ל־Phase 3: משימות **בלי אודיו** ובלי **כתיבה חופשית ארוכה**, תוך שימוש בממשק הקיים (MCQ / הקלדה מבוקרת לפי `answerMode`).

## עקרונות

- תשובה **מוגבלת**: מילה אחת, משפט קצר אחד, או בחירה מובנית בין 4 ניסוחים.
- **ציון דטרמיניסטי**: `acceptedAnswers` / נרמול ל־typing; MCQ עם מפתח ברור.
- **אותנטיות**: הקשר חברתי/לימודי אמיתי, לא רק זיהוי צורה; עדיף טקסט מצב קצר מ״שאלה טכנית״ בלבד.

## רשימת `authenticity_pattern` (ערכים מומלצים)

| pattern | תיאור | דוגמת שימוש |
|--------|--------|-------------|
| `micro_passage_main_idea` | שני משפטים + בחירת רעיון מרכזי | הבנת הנקרא |
| `micro_passage_link_theme` | שני משפטים + מה משותף | הבנת הנקרא |
| `single_sentence_cause_choice` | משפט אחד + הסבר סיבה מבין ברירות | היסק קל |
| `guided_description_sentence_choice` | הנחיה לתיאור, בוחרים משפט מתוך 4 | דיבור (תת־נושא describe) |
| `situational_phrase_register` | ניסוח מתאים למצב (כיתה/נימוס) | דיבור / situation_register |

השדה `authenticity_pattern` אופציונלי על פריט בבנק; כשקיים, הוא מועבר ל־`params` של השאלה בזמן ריצה (לא משנה UI).

## הרחבה עתידית (במסגרת אותה גדר)

- **התאמת סדר** (3–4 משפטים) כ־MCQ בלבד.
- **תיקון מינימלי** (בחירת משפט מתוקן מתוך 4) — לא עריכה חופשית.
- **השלמת חסר מילה אחת** בטקסט קצר — רק אם `answerMode`/גייט מאשרים הקלדה מבוקרת.
