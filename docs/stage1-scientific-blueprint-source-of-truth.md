# Stage 1 — Scientific Blueprint / Diagnostic Source of Truth

**סוג מסמך:** מקור אמת ארגוני מחייב (מסגרת אבחון ופלט)  
**שפה:** עברית  
**גרסה:** 1.0 (מועמד להקפאה)  
**Baseline פנימי:** v0.9 — איחוד טיוטות ו־Tightening סופי  
**תאריך ייחוס:** 2026-04-14  

**גבולות מסמך (מחייב):** אין כאן יישום קוד, UI, PDF, מנוע, שדות payload, או probes בפועל. המסמך מגדיר שפה, כללי החלטה, טקסונומיה, ראיות, ביטחון, עדיפות, ושערי פלט.

## 1. מטרת המערכת

### 1.1 מה המערכת כן

מנוע תמיכה בהחלטות **פדגוגיות** מבוססות התנהגות במשימות דיגיטליות: איסוף דפוסי ביצועים, דירוג **השערות אבחוניות מוגבלות**, דירוג **ביטחון**, הצעת **probe** להכרעה בין הסברים מתחרים, והמלצה על **כיוון התערבות** (לא טיפול קליני).

### 1.2 מה המערכת לא

| תחום | המערכת אינה |
|------|-------------|
| קליני | אינה מאבחנת לקויות למידה, ADHD, חרדה, דיסלקציה וכו׳ |
| הערכה רשמית | אינה מחליפה מבחן, ציון בית ספר, תעודה |
| גורם מחייב | אינה קובעת «מה הילד באמת» מחוץ למה שנצפה במשימות |
| גידול משפחתי | אינה מנחה משמעת, מסכים, שינה |

### 1.3 משמעות «אבחון»

**אבחון** = תווית פדגוגית **מותנית בראיות**, המתארת דפוס קושי או חוזק **יציב** ברמת **תת־מיומנות (subskill)** בתוך מקצוע, עם **הסבר שורש אפשרי** והפרדה מפורשת מ**הסברים מתחרים** ברמת הביטחון הנתונה. ברירת המחדל: האבחנה נקבעת על **צבר subskill** (ראו סעיף 3).

### 1.4 משמעות «רמת ביטחון»

כלל מחייב על **מה מותר לומר**, האם מותרת **המלצת התערבות**, והאם חובה **probe** לפני מסקנה מעשית. ראו סעיפים 9, 10, 11.

### 1.5 משמעות «early signal»

אינדיקציה **חלשה אך עקבית** לכיוון סיכון/הזדמנות — **ללא** מספיק ראיות לקביעה יציבה. **חובה:** ניסוח מוגבל + probe או מעקב; **אסור:** תוכנית שינוי ארוכת טווח כאילו הוכח.

### 1.6 מתי המערכת תומכת בהחלטה

כאשר **שערי הפלט** (סעיף 11) מתירים diagnosis או intervention, וקיימות **ראיות מינימום** לפי הטקסונומיה, **ללא סתירה חזקה**, וביטחון ≥ בינוני לאותו סוג פלט.

### 1.7 מתי נדרש איש מקצוע אנושי

| מצב | חובת אנוש |
|-----|-----------|
| קושי חוזר חריף במספר מקצועות + ירידה חדה בזמן קצר | מורה / יועץ בית ספר |
| מצוקה / התנגדות (לפי דיווח מבוגר) — מחוץ למנוע | איש מקצוע מתאים |
| דרישת «אבחנה סופית» כשהביטחון נמוך או הראיות סותרות | אסור למנוע לספק אבחנה סופית |
| חשד לבעיה ראייה / שמיעה / מוטוריקה (מבוסס דיווח) | אבחון חיצוני |

## 2. מסגרת אבחון ליבה — ארבע שכבות

### 2.1 שכבת אבחנה (Diagnosis)

| רכיב | תוכן מחייב |
|------|------------|
| הגדרה | תיאור דפוס קושי/חוזק ברמת subskill בהקשר משימתי. |
| פלטים מותרים | קטגוריה מתוך הטקסונומיה; ניסוח מותנה; ציון סוגי ראיה (לא ערכי מוצר). |
| אסור | קיבוע תווית לילד; איחוד מקצוע ל«חלש»; הסקת מוטיבציה/אינטליגנציה. |
| קשר לראיות | כל תווית חייבת מסלול הוכחה: מינימום + חזקה אופציונלית + «לא להסיק». |
| הקשחה | פלט אבחנה **חייב** מזהה צבר: `subject` + `topic` + `subskill` + חלון זמן/אירועים, או נימוק מפורש לאגרגציה למעלה. |

### 2.2 שכבת ביטחון (Confidence)

| רכיב | תוכן |
|------|------|
| הגדרה | כללים לרוחב ראיות, יציבות, הפרדת מתחרים. |
| פלטים מותרים | נמוך / בינוני / גבוה / early signal בלבד / נתונים לא מספיקים / סתירתי. |
| אסור | «בטוח ב־100%»; העלאת ביטחון מאירוע בודד חריג. |

### 2.3 Next best probe (הגדרה במסמך)

משימת הבחנה מכוונת שמטרתה להפריד בין שתי השערות או לאמת subskill יסוד. אינה ממומשת כאן בקוד.

### 2.4 המלצת התערבות (Intervention)

כיוון פדגוגי: פעולה מיידית + תרגול קצר מובנה + מה להימנע + סימני שיפור/כישלון. **מותרת** רק אם סעיף 11 מתיר.

## 3. יחידת ניתוח, רמת קביעה, ושרשרת צבר

### 3.1 יחידת האבחון הראשית

| רמה | יחידה | תפקיד |
|-----|--------|--------|
| **ראשית (ברירת מחדל)** | **צבר אירועים על subskill בתוך topic** | כאן נקבעת **אבחנה מוצעת** או «לא ניתן להסיק». |
| משנית | משימה בודדת | **תרומת ראיה** בלבד; לא אבחנה סופית בכוחה לבדה. |
| עליונה | דפוס מקצועי (subject pattern) | סיכום אגרגטיבי; **מותר** רק עם ראיית **רוחב** (סעיף 10) ושער פלט (סעיף 11). |

### 3.2 באיזו רמה נקבעת אבחנה

1. **שאלה בודדת** → תצפית (נכונות, זמן, ניסיונות, רמזים) ממופה ל־**subskill**.  
2. **Subskill** → צבר לפי **כללי ספירה** בטבלאות סעיף 5 (מתי נספר / לא נספר, תדירות מינימום).  
3. **Topic** → ארגון קוריקולרי; אבחנה מוצעת בדרך כלל **מתחת** לרמת topic (subskill), אלא אם יש רוחב חוצה־subskills בתוך אותו topic.  
4. **דפוס מקצועי** → רק כשאותו **סוג קושי לוגי** חוזר ב**מספר topics** או כש**חוסר יציבות גלובלי** מחייב סיכום — עם סעיפים 10–11.

### 3.3 שרשרת מיפוי

```
שאלה בודדת → topic (מפתח נושא) → subskill → צבר תצפיות (חלון + תדירות מינימום)
  → אבחנה מוצעת + ביטחון (§9) + עדיפות (§10) → שער פלט (§11)
```

**כלל מחייב:** מעבר מ־subskill ל־**topic שלם** או **מקצוע שלם** דורש ראיית **רוחב** + עמידה ב־§10–§11.

## 6. התקדמות למידה ומפת תלות (תמצית)

**מתמטיקה:** ספירה יציבה → ייצוג מספר → עובדות חיבור/חיסור → נשיאה → כפל/חילוק → שברים (ייצוג → המראה) → עשרוניים → מילולי רב־שלבי.  
**טעות נפוצה:** קושי בשברים שנראה כשברים — לעיתים פער **כפל/פירוק**; מילולי — **יחידות** או **ניהול שלבים**.

**גאומטריה:** צורות בסיסיות → זוויות → מקביליות/מאונכות → שטח מלבן → מקבילית/משולש → נפח.  
**בלבול:** שטח מקבילית עם מספרים נכונים — לעיתים **בחירת גובה**, לא כפל.

**עברית:** פענוח בסיסי → אוצר → דקדוק התאמה → כתיב שיטתי → ניקוד הקשרי → הנקרא מורכב.  
**בלבול:** ניקוד — לעיתים **אוצר** (הומופונים).

**אנגלית:** אוצר בסיס → סממני זמן → מבני משפט → קולוקציות/קריאה מורכבת.  
**בלבול:** דקדוק — לעיתים **תרגום מילולי** מאוצר עברי.

**מדעים:** תצפית מול הסבר → סיווג → משתנה מבודד → מודל מערכת.  
**בלבול:** כישלון בניסוי בגלל **קריאת טקסט**.

**מולדת וגאוגרפיה:** סימון בסיסי → מפתח מפה → סולם → כיוון → ציר זמן/אזרחות.  
**בלבול:** טעות «גאוגרפית» מ־**שם דומה**.

## 10. מסגרת עדיפות וחומרה (Priority / Severity)

| רמה | שם | משמעות |
|-----|-----|--------|
| P1 | עדיפות נמוכה | שימור / מעקב |
| P2 | עדיפות בינונית | מיקוד מוגבל; probe אופציונלי |
| P3 | עדיפות גבוהה | מיקוד ממושך; probe או התערבות לפי §11 |
| P4 | סקירה דחופה | דחיפות פדגוגית־ארגונית: חוסר התקדמות חוזר, רוחב חריג, או ירידה חדה — **חובה** מעורבות מבוגר; **לא** אבחון קליני |

**נגזרות:** עמידה במינימום חזרתיות בטבלאות §5 + סמנים חזקים מול נגד־ראיה חלשה → מעלה רמה; נגד־ראיה חזקה → מורידה. אותו דפוס לוגי ב־**2+** topics או **2+** subskills לא קשורים → עלייה בדרגה אחת לפחות. חזרה ב־**2** חלונות זמן (סשנים/ימים) → עלייה; טעות חד־פעמית → נשאר נמוך/בינוני לפי שאר הכללים.

**מטריצת החלטה (מחייבת):**

| ביטחון | רוחב צר (subskill אחד) | רוחב בינוני (topic) | רוחב רחב (מספר topics / מקצוע) |
|--------|-------------------------|----------------------|----------------------------------|
| גבוה | P3 | P3 | P4 |
| בינוני | P2 | P3 | P3 |
| נמוך / early | P1 | P2 | P2 |
| נתונים לא מספיקים | P1 | P1 | P2 |
| סתירתי | P2 | P3 | P4 |

**כלל P4:** מחייב גם אחד מאלה: (א) ירידה חדה ביציבות ב־≤7 ימים או 3 סשנים; (ב) כישלון חוזר אחרי שני מחזורי התערבות מובנים; (ג) סתירה חוצת־מקצועות עם אותו סמן נצפה.

## 11. שערי פלט (Output gating)

| תנאים (כל התנאים בשורה) | פלט מותר |
|--------------------------|----------|
| מינימום חזרתיות + «נספר כאשר» + ביטחון בינוני ומעלה + אין נגד־ראיה חזקה יחסית | **Diagnosis** (תווית + ניסוח מותנה) |
| ביטחון בינוני ומעלה אך חסר רוחב מינימלי או הפרדה מלאה בין מתחרים | **תיאור דפוס + ביטחון** בלבד, או Diagnosis עם סייג «מדגם צר» |
| ביטחון נמוך / early signal / רוחב לא מספיק אך יש כיוון | **Next probe בלבד** (+ confidence כטקסט) |
| ביטחון גבוה + P≥P2 + אין סתירה + §10 לא חוסם | **Intervention** בנוסף ל־diagnosis |
| ביטחון גבוה + P3–P4 + מינימום חזרתיות + אין נגד־ראיה חזקה | **Intervention מלא** לפי §12 |
| נתונים לא מספיקים או כל האירועים תחת «לא נספר כאשר» | **Cannot conclude yet** — חובה; אסור diagnosis |
| סתירת ראיות | **Cannot conclude yet** + הצגת סתירה; אסור intervention; חובה probe |
| P4 | מותר diagnosis/confidence/probe לפי העילה; **חובה** ניסוח שמפנה לסקירה עם מבוגר/מורה; **אסור** לסגור תיק לבד |

**טבלת סיכום שערים:**

| מצב | Diagnosis | Confidence בלבד | Probe בלבד | Intervention | Cannot conclude |
|-----|-----------|-----------------|--------------|--------------|-----------------|
| מדגם צר + בינוני | מותר עם סייג | חלופה | אם נדרש | לא | לא |
| נמוך / early | לא | כן | חובה עיקרי | לא | אם אין אירועים מספיק |
| גבוה + P2+ | כן | לא נדרש | אופציונלי | מותר ממוקד | לא |
| סתירה / לא נספר | לא | חלקי | חובה | לא | חובה |
| P4 | לפי שורות למעלה | כן | כן | רק אם ביטחון גבוה ואין סתירה | אם יש סתירה |
