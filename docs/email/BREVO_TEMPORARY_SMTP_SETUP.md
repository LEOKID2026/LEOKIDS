# Brevo Temporary SMTP Setup — Launch Path

**Project:** Hebrew learning site — External Email Delivery  
**Provider:** Brevo Free via Supabase Custom SMTP  
**Audience:** Owner only — all configuration steps require dashboard access  
**Status:** Temporary launch path until a custom domain is purchased and Resend (or similar) is configured with SPF/DKIM/DMARC

---

## Why Brevo for Temporary Launch

| Factor | Detail |
|--------|--------|
| **Problem** | Supabase default SMTP is limited to ~2 emails/hour — insufficient for launch Auth flows (password reset, signup confirmation, admin password-setup). |
| **SMTP2GO considered** | Signup blocked when using a Gmail address; required an email at a private domain. |
| **Brevo chosen** | Free account created successfully; Transactional email API/SMTP available without a purchased custom domain for initial launch. |
| **Free plan limit** | **300 emails/day** — sufficient for small launch volume; monitor usage in Brevo Dashboard. |
| **Auth trigger** | Supabase Auth remains the trigger. Only the SMTP delivery path changes via Custom SMTP. |
| **App code** | No application code changes required (confirmed by audit). |

This is a **temporary launch setup**. When a custom domain is purchased, migrate to Resend with verified domain DNS (see [`RESEND_SETUP_CHECKLIST.md`](./RESEND_SETUP_CHECKLIST.md)).

## What NOT to Do

| Do not | Reason |
|--------|--------|
| Paste Brevo SMTP key in chat, git, or `.env` | Secret — Dashboard and password manager only |
| Add `BREVO_*` or SMTP keys to `.env.example` | App does not send email directly |
| Use Vercel hostname as email sending domain | Not suitable for production email authentication |
| Treat this as permanent production email setup | Migrate to Resend + custom domain when ready |

## Migration Path (When Custom Domain Is Ready)

1. Purchase and connect custom domain
2. Set up Resend with `mail.<yourdomain>` or `auth.<yourdomain>` (see [`RESEND_SETUP_CHECKLIST.md`](./RESEND_SETUP_CHECKLIST.md))
3. Update Supabase Custom SMTP from Brevo to Resend
4. Re-run live tests; update QA report long-term track (L-01–L-08)
